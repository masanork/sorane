# Multilingual / hreflang (Phase 3)

Static-site i18n for sorane: locale-prefixed content paths, per-page `lang`, and
`hreflang` / `og:locale:alternate` in `<head>`.

## Configuration (`sorane.yaml`)

```yaml
site:
  lang: ja
  base_url: https://www.example.go.jp/
  i18n:
    default: ja          # optional; defaults to site.lang
    locales:
      en:
        lang: en
        path_prefix: en  # single segment; no slashes
```

- **Default locale** — content at `content/` root (e.g. `content/about.md` → `about.html`).
- **Other locales** — mirror paths under `content/{path_prefix}/` (e.g. `content/en/about.md` → `en/about.html`).
- Omit `site.i18n.locales` (or leave empty) for single-language sites (unchanged behaviour).

## Content

### Mirrored paths

| Source | Output |
|--------|--------|
| `content/about.md` | `about.html` |
| `content/en/about.md` | `en/about.html` |
| `content/index.md` | `index.html` (blog index, default locale) |
| `content/en/index.md` | `en/index.html` (locale blog/list index) |

### `translation_key` (optional)

When filenames differ across locales, group alternates with frontmatter:

```yaml
translation_key: about-us
```

## Build output

| Feature | Behaviour |
|---------|-----------|
| `<html lang>` | Per-locale / per-page (`frontmatter.lang` overrides) |
| `<link rel="alternate" hreflang>` | Sibling translations + `x-default` (default locale) |
| `og:locale` / `og:locale:alternate` | From page lang and siblings |
| JSON-LD `inLanguage` | Per-page lang |
| `sitemap.xml` | All locale URLs |
| Blog archives / tags | Default locale only (Phase 3 scope) |

## Validation

No new validate categories in Phase 3. Missing translation siblings are not errors.
